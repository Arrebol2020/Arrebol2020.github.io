马尔可夫决策(MDP)

在马尔可夫决策决策过程中，它的环境是全部可以观测的(fully observable)。但是很多时候环境里面有些量是不可观测的，但这个部分观测的问题也可以转化一个 MDP (马尔可夫决策)的问题

马尔可夫决策过程的基础

  • 马尔可夫过程(Markov Processes)
  • 马尔可夫奖励过程(Markov Reward Processes)

马尔可夫过程

如果一个状态转移是符合马尔可夫的，那就是说一个状态的下一个状态只取决于它的当前状态，而跟它当前状态之前的状态都没有关系。

马尔可夫性质：未来的转移跟过去是独立的，它只取决于现在。该性质是马尔可夫过程的基础

状态转移矩阵

状态矩阵类似于一个条件概率，当知道当前状态 s_t 过后，到达下面所有状态的一个概念。它的每一行描述了 从一个节点达到所有其它节点的概率。状态转移矩阵如下图：

马尔可夫奖励过程

马尔可夫奖励过程(Markov Reward Process, MRP)是马尔克夫链再加上一个奖励函数(Reward Function)和一个折扣因子(Discounted Factor)。奖励函数是一个期望，就是当到达某一个状态的时候，可以获得多大的奖励。

理解 MRP ：可以把它看成一个纸船，我们把一个纸船放到河流之中，那么它就会随着这个河流而流动，它自身是没有动力。当我们从某一点开始的时候，这个纸船就会随着事先定义好的状态转移进行流动，它到达每个状态过后，我们就有可能获得一些奖励

折扣因子(Discounted Factor)

  • 有些马尔可夫过程是带环的，它并没有终结，折扣因子可以避免这个无穷的奖励
  • 我们想把这个不确定性表示出来，希望尽可能快的得到奖励，而不是在未来某一个点得到奖励
  • 如果这个奖励是有实际价值的，我们可能更希望立刻就得到奖励，而不是后面再得到奖励
  • 大家都想得到即时奖励
  • 在有些时候，这个系数可以设置为 0 ，表示我们只关注它当前的奖励。也可以把它设置为 1 ，表示对未来并没有折扣，未来获得的奖励跟当前获得的奖励是一样的

MRP的例子

对于 s4, s5, s6, s7 轨迹的奖励进行计算，折扣因子设为 0.5

  • 1 在 s4 的时候，奖励为 0
  • 2 下一个状态 s5 的时候，折扣系数为 1/2
  • 3 s6的折扣系数 1/4
  • 4 s7的折扣系数为 1/8
    所以对于这个轨迹，它的 return 为1.25.类似的可以得到其他轨迹的 return

计算当前状态的价值

一个可行的做法就是说我们可以产生很多轨迹，然后把这里的轨迹都叠加起来。比如从 s4 开始，采样生成很多轨迹，都把它的 return 计算出来，然后可以直接把它取一个平均作为 s4 的价值

贝尔曼等式(Bellman Equation)求解

适用于很小量的MRP

这里采取了另外一种计算方法，通过一定的推导就可以从这个价值函数里面推导出贝尔曼等式(Bellman Equation)，它定义了当前状态跟未来状态之间的关系

贝尔曼等式定义了状态之间的迭代关系。

迭代方法

  • 动态规划(Dynamic Programming)
  • 门特卡罗(Monte-Carlo evaluation)
  • 时序差分学习(Temporal-Difference learning)：动态规划和门特卡罗结合
动态规划算法

通过 boot straping 的方法，不停地迭代这个贝尔曼等式。当这个最后更新的状态跟上一个状态变化并不大的时候，这个更新就可以停止，我们就可以输出最新的V’(s)作为它当前的状态

门特卡罗算法

当得到一个 MRP 过后，我们可以从某一个状态开始，然后让它把这个小船放进去，让它随波逐流，这样就会产生一个轨迹。产生了一个轨迹过后，就会得到一个奖励，当累积到一定轨迹数量过后，直接除以这个轨迹，就可以得到它的这个价值

比如要计算 s4 状态的价值，就可以从 s4 状态开始，随机产生很多轨迹，就产生很多小船，然后扔到这个转移矩阵里面去，它就会随波逐流，产生轨迹。每个轨迹，我们可以计算得到它的 return。每个轨迹都会得到一个 return，得到大量的 return，然后直接取一个平均，就可以等价于它现在s4的价值

马尔可夫决策(MDP)

相对于马尔可夫奖励过程，马尔可夫决策过程多了一个决策。

决策(Policy)

Policy 定义了某个状态应该采取什么行为。当知道当前状态后，可以代入到决策函数(Policy Function)后，会得到一个概率，概率代表了在所有可能的行为里面怎样去采取行动。如有 0.7 的概率往左走，有 0.3 的概率往右走，这样是一个概率的表示。另外这个策略也可能是确定的，它有可能是直接输出一个值，或者就直接告诉你当前应该采取什么样的行为，而不是一个行为的概率。

MP/MRP 和 MDP 的比较

  • 马尔可夫过程的转移是直接就决定
  • 马尔可夫决策中间多一层行为 a ， 就是当你在这个状态的时候，首先要决定的是采取哪一种行为，那么就会到达了某一个黑色的节点。到了这个黑色节点，因为有一定的不确定性，当你当前状态决定过后以及当前采取的行为过后，你到未来的状态其实也是一个概率分布。所以在这个当前状态跟未来状态转移过程中这里多了一层决策性

MDP 的价值函数

  • 它的定义和 MRP 是类似的，但是这里是期望是基于决策的，当 policy 决定过后，通过这个 policy 进行采样来得到一个期望，那么就可以计算出它的这个价值函数
  • 引入了 Q 函数(action-value function)。它定义的是在某一个状态采取某一个行为，然后它有可能得到的这个 return 的一个期望。这里期望其实也是 over policy function。所以你需要对这个 policy function 进行一个加和，然后后得到它的这个价值。
  • 对 Q 函数中的行为进行加和，就可以得到价值函数

Backup 图

Backup 类似于 boot straping 之间的迭代关系，就对于某一个状态，它的当前这个价值是跟它未来价值线性相关的。如上图，第一层加和就是这个叶子节点，然后往上一层的话，我们就可以把未来的这个价值 s’ backup 到黑色的节点。然后再有一个层加和，第二层加和是把 action 进行加和。得到黑色节点的价值过后，再往上 backup 一层，然后就会推到根节点的价值，根节点就是我们当前状态。所以 Backup Diagram 定义了未来下移时刻的状态跟上一时刻的状态之间的一个关联。

同样对于 Q 函数，也可以进行这样的一个推导，就现在的根节点是这个 Q 函数的一个节点。这个 Q 函数是对于黑色的这个节点。我们下一时刻的这个 Q 函数是叶子节点，有四个黑色结点。那么我们这里也有两个加和。第一层加和是先把这个叶子节点从黑节点推到这个白色的这个节点，进了它的这个状态，就当我们到达某一个状态过后，这个白色节点，然后再进行一个加和，这样 就把它重新推回到当前节点的一个 Q 函数，所以这个等式就决定了未来 Q 函数跟当前 Q 函数之间的这个关联。
