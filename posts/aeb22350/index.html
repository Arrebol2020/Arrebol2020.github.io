论文笔记 Classification is a Strong Baseline for Deep Metric Learning

发表于2020-11-19|更新于2021-04-07|论文论文笔记
|字数总计:2.2k|阅读时长:6分钟|阅读量:

简介

这篇论文，作者主要研究了 classification-based 方法在图片检索的数据集中的效果，在例如 CAR-196, CUB-200-2011, Stanford Online Product 和 In-shop 数据集等几个标准的检索数据集中评估了图像检索和聚类，同时建立了他们自己的 classification-based 方法，该方法在不同的特征大小和基础的特征网络中都具有很好的表现。之后，作者进一步深入研究了基于可伸缩分类训练 (scalable classification-based training) 的子采样类的性能影响，以及二值化的影响，为实际应用提供有效的存储和计算，同时提出一种 classification-based 的方法，用于学习高维的 binary embeddings，实现了先进的检索性能相同的，内存占用 64 维 float embedding 的检索任务

深度度量 (学习图像嵌入) 在人脸验证、fine-grained 检索、聚类和视觉检索中是一个核心问题。标准的深度神经网络通过以 pair 和 triplets 形式的图像之间的局部关系来学习图像映射。两大应用：

  • 基于内容的图像检索 (content-based image retrieval) ：当前主流方法为： triplet-based nonparametric training
  • 人脸验证 (face verification) ：当前主流方法为： classification-based parametric training

其学习目标：在开放式设置机器学习的测试期间，所学的嵌入能够很好地描述新实例。学习一个将图片像素映射成嵌入特征向量的函数，即学习一个好的嵌入空间，使样本之间的相似性作为样本嵌入向量之间的距离，用于评估两张图片的相似度。在实际应用中，部署的机器学习系统需要处理大量没见过的数据，这一个目标与此非常一致

度量学习的核心挑战是需要抽取信息样本用于训练（过于困难的消极因素会破坏训练的稳定性，而过于简单的消极因素会导致 triplets 的损失近乎为零，从而导致收敛速度缓慢）。解决方法：

  • 利用一批的所有图片的关系来形成 informative triplets (这些方法需要 batch 足够大，确保能够形成 informative triplets)实际上， batch 的大小受限于物理内存的大小，因此随着数据集大小的增长，仍然面临概率递减的挑战。
  • 使用 classification-based training：与 triplet-based 方法不同，该方法可以被视为使用 proxy 近似每个类，并使用所有的 proxy 来为每次训练迭代提供全局上下文。虽然 classification-based 的度量学习简化了训练，但是由于去掉了采样，因而在 extreme classification 问题中具有可伸缩性的限制。同时，其参数性质在 fine-grained open-set 识别设置中可能会造成问题

    在 extreme classification 训练中，拓展到更多的 class 被认为是不切实际的。

Training Sampling

  • Semi-hard sampling in conjunction with triplet-loss：目前已被广泛的应用于许多任务中
  • Distanced-weighted sampling：该方法表明在训练过程中困难均衡混合的情况下，图像检索性能可以进一步提高
  • Hierarchical Triplet Loss：提出了通过在训练过程中动态将类似的 class 合并到一个层次数中，可以从这个结构中得到更多的信息样本，并且这种损失也为训练提供了全局上下文

Ensembling Embedding

集成嵌入 (ensembling embedding) 是进一步提高图像检索性能的最新焦点，通过增强，倾向于不同的空间位置或者分割训练类别来训练。然而，这样的集成嵌入牺牲了更高维度的图像检索性能，同时还向系统中引入了额外的超参数

作者发现维数与检索性能直接相关。更高维度的嵌入可以提高检索性能，但是低维嵌入对于可伸缩性来说是首选，可以减少存储和距离计算成本，特别是在大型应用程序中，如视觉检索。但是作者注意到，当增加嵌入的维数时，优化器并没有充分利用更高维的度量空间，相反，看到特征维度开始更少地依赖于每个特征维度的大小，转而去依赖符号值。

在图 2 中，可以看到两个数据集的 Recall@1 二进制特性的性能 (将特性值阈值设置为 0 ) 与浮点嵌入的性能收敛到一起。这是在数据集合模型架构中看到的一致结果。作者展示了训练高维嵌入和二值化可以在性能和可伸缩性之间实现最佳的平衡

Normalized Softmax Loss

在训练分类网络进行度量学习时，作者去除最后一个线性层中的偏置项，并在 softmax 之前，将输入和权重进行 L2 正则化来优化余弦相似度 (cosine similarity) 。Temperature scaling 用于放大 class 之间的差异和提高梯度。公式如下：

当我们将 class 的权重视为 proxy，并选择距离度量作为余弦距离函数时，标准化 softmax loss 就符合代理范式 (proxy paradigm)。一种更普遍的 classification loss，LMCL (Large Margin Cosine Loss) 已经被提出用于一个额外的边缘超参数用于人脸验证

Layer Normalization

在特征模型的最终池化层之后，立即添加不带反射函数的 Layer Normalization，有利于对嵌入的特征维数进行归一化，使其值的分布中心为零。这样我们可以通过在 0 处设定阈值来轻松地对嵌入进行二值化。Layer normalization 有助于网络更好的初始化新参数，达到更好的优化

作者利用不带参数的 Layer Normalization 来对池化后的激活值进行标准化，用以初始化训练。在训练中加入 Layer Normalization 允许在跨模型架构的新参数的 poor weight initialization 保持稳健，从而减轻了谨慎初始化权重的需求

Class Balanced Sampling

基于对使用分类进行度量学习的 proxy 解释，其损失受限于 class 中最坏的近似例子。解决这个问题的一个简单方法是在构建训练 mini-batch 时，增加每个类中的样本。对于每个训练批次，我们首先对 C 类进行抽样，然后对每个类内的 S 类进行抽样。

与数据集上的随机顺序迭代相比，类平衡抽样是十分有益的。通常，当每个类拥有更多的样本时，性能会得到提高，直到在 mini-batch 中存在太少的不同分类，分离少数不同类的偏差可能会给优化带来过多的干扰，从而导致性能下降

Implementation

  • 使用 ImageNet ILSVRC-2012 的预训练权重来初始化网络
  • 在每个架构中的 pool5 中添加一个随机初始化的全连接层，用于学习不同尺度的嵌入
  • 为了减轻对全连接层初始化的敏感性，在 pool5 和全连接层之间条件了一层 Layer Normalization
  • 在 Softmax 之前对嵌入权重和类权重进行 L2 正则化，并始终使用 0.05 的 temperature
  • 除非另有说明，首先训练一个 epoch ，只更新新的参数，以更好地进行初始化
  • 在之后的 30 epoch 中，优化所有参数， batch 大小为 75 ，学习率为 0.0 (CUB200 除外，在 CUB200 中学习率设置为 0.001 ，用于防止过拟合)
  • 在构建 batch 时，对于 Cars196 和 CUB200 每个类抽取 25 个样本，对于 SOP 和 InShop 每个类抽取 5 个样本 (数据集中每个类仅抽取 5 张图像)
  • 在 15 epoch 时，使用动量为 0.9 的 SGD 方法，权重衰减为 1e-4，γ 为 0.1 用于减小学习率 α
  • 在训练中，对于 256x256 的图像，使用水平镜像和随机裁剪的方法；在测试期间，从256x256的图像中进行中央裁剪。之后，对于 GoogleNet 的裁剪成 227x227 ，其它裁剪成 224x224

Subsampling for Classification Scalability

通过子抽样，对于每个训练 mini-batch ，网络只需要在所有类的子集 (随机抽样的子集，其中包括训练 mini-batch 中的所有类) 中进行分类

作者表明训练过程中对类进行子抽样是一种有效的 softmax 嵌入训练方法，其性能下降很小
