理论

经典的特征 descriptors (SIFT, SURF, …) 通常使用 Euclidean distance 或 (L2-norm) 进行比较和匹配 。因为 SIFT 和 SURF 的 descriptors 代表了一个邻域的梯度方向直方图 (SURF 的 Haar wavelet response)，Euclidean distance 的替代方法是 histogram-based metrics (X^2，Earth Mover’s Distance (EMD), …)

Arandjelovic et al 提出了 RootSIFT descriptor

使用平方根 (Hellinger) kernel 来替代标准的 Euclidean distance 来度量 SIFT descriptors 之间的相似度，从而在整个流程的各个阶段都有显著的性能提升

Binary descriptors (ORB, BRISK, …) 使用 Hamming distance 。这个距离相当于计算二进制字符串中不同元素的数量 (使用了 XOR 运算后的 population count)

dhamming(a,b)=∑i=0n−1(ai⊕bi)d_{hamming} \left ( a,b \right ) = \sum_{i=0}^{n-1} \left ( a_i \oplus b_i \right )

为了对匹配进行过滤，Lowe 提出使用距离比 (distance ratio) 检测来消除错误匹配。计算 considered keypoint 的两个最接近匹配之间的距离比，当这个值低于设定的阈值时，它就是一个好的匹配。经过实验，这个比率可以帮助区分模糊的匹配 (最近的两个 neighbors 的距离比接近于 1) 和易于区分的匹配。下图来自 SIFT 论文 ，说明了基于最近邻距离比检测的匹配正确的概率

代替或额外的过滤器检测有：

  • cross check test： 交叉检查检验，若特征 f_b 是 I_b 中 f_a 的最佳匹配，特征 f_a 是 I_a 中 f_b 的最佳匹配，则称 (f_a, f_b) 是一个好的匹配
  • geometric test：几何检验，消除不适合几何模型的匹配，例如：planar objects 的 RANSAC 或 robust homography

cv::FlannBaseMatcher

Flann-based descriptor 匹配器。该匹配器从一个 train descriptor 集合上训练 cv::flann::index ，并调用它最近的搜索方法来找到最佳匹配。FlannBasedMatcher 不支持屏蔽允许的 descriptor 集匹配。因为 flann::index 不支持这样做

Clustering and Search in Multi-Dimensional Spaces

FLANN (Fast Library for Approximate Nearest Neighbors)：是一个包含了用于在大数据集中快速最近邻居搜索和高维特征的一组算法库
