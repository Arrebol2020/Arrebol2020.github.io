过拟合(overfitting)与欠拟合(underfitting)

  • 最左边的图像表明算法没有很好地拟合训练集数据。因而是欠拟合(underfitting)/具有高偏差(high bias)
  • 中间的图像拟合效果很好
  • 最右边的图像，似乎很好地拟合了数据集，以为它通过了所有的数据点，但这是一条扭曲的曲线，过度拟合了训练集的数据。因而是过拟合(overfitting)/高方差(high variance)

过拟合：会在变量过多的时候出现，这时训练的假设函数能够很好地拟合训练集，所以它的代价函数可能非常接近 0 ，但是它无法泛化到新样本，即泛化能力极差

解决过拟合问题

  • 减少特征的数量
    • 人工检查变量清单并以此决定哪些变量更为重要，哪些特征变量应该保留，哪些变量应该舍弃
    • 使用模型选择算法，该算法能自动选择哪些变量应该舍弃，哪些变量应该保留
  • 使用正则化(regularization)
    • 保留所有的特征数量，但是减少量级或参数 θ 的大小，该方法十分有效，当我们有很多特征变量时，其中每个变量都能对预测的 y 值产生一点影响

正则化(regularization)

如果我们的参数值较小，即会产生一个更简单的假设模型，得到的函数就会越平滑，也更不容易出现过拟合的问题

注意添加的正则化项的 j 的值是从 1 开始，而不是从 0 开始。从 0 开始也可以，但大部分还是从 1 开始

λ (regularization parameter，正则化参数)的作用：

  • 控制两个不同目标之间的 trade-off。一个目标是更好的拟合训练集数据；另一个目标是让参数尽可能地小
  • 即更好地区拟合训练集的目标和将参数控制得更小的目标，从而保持假设模型的相对简单，避免出现过拟合的问题
  • 当 λ 设置过大，将会导致 θ 的值趋于 0，从而使得模型欠拟合

线性回归的正则化

θ_j 的第一项 (1-αλ/m) 总是比 1 小一点，所以该项会将 θ_j 向 0 的方向缩小。第二项并没有发生任何变化，完全与在添加正则化项之前的梯度下降更新一样

正规方程的正则化

添加了 λ × 一个除了最左上角的元素都是 0 外，其余的主对角线元素均为1，剩下的元素均为 0 的矩阵。进行正则化还可以解决一些 X^TX 出现不可逆的问题
