Features2D + Homography 来寻找已知的目标

发表于2021-01-16|更新于2021-04-20|教程OpenCV
|字数总计:2.2k|阅读时长:10分钟|阅读量:

目标

在这个教程，你将会学习如何：

  • 使用函数 cv::findHomography 来找到匹配关键点之间的变换
  • 使用函数 cv::perspectiveTransform 来映射这些点

因为要使用 SURF ，目前该算法被专利保护了，可通过编译来使用 SURF 。教程链接：

理论

cv::findHomography

找到两个平面之间的透视变换，用于寻找初始化的 extrinsic matrices 。在一定尺度下确定 Homography matrix 。因此，将其标准化，使

h33=1h_{33}=1

若一个 H 矩阵无法估计时，将返回一个空矩阵

cv.findHomography(srcPoints, desPoints, method=0, ransacReprojThreshold=3, mask, maxIters=2000, confidence=0.995)

  • srcPoints：原平面上点的坐标，类型为 CV_32FC2 或 vector 的矩阵
  • desPoints：目标平面上点的坐标，类型为 CV_32FC2 或 vector 的矩阵
  • method：用于计算 homography 矩阵的方法。主要为：
    • 0：一种使用所有点的常规方法，即最小二乘法
    • RANSAC：RANSAC-based 方法
    • LMEDS：Least-Median 方法
    • RHO：PROSAC-based 方法
  • ransacReprojThreshold：
  • mask：可选输出掩码，由方法( RANSAC 或 LMeDS )设置。注意，输入掩码值将被忽略
  • maxIters：RANSAC 方法的最大迭代次数
  • confidence：置信水平，在0到1之间
    该函数查找并返回原平面和目标平面之间的透视转化 Hsi[xi′yi′1]∼H[xiyi1]s_i \vecthree{x'_i}{y'_i}{1} \sim H \vecthree{x_i}{y_i}{1}

这样 back-projection 误差可以被最小化

∑i(xi′−h11xi+h12yi+h13h31xi+h32yi+h33)2+(yi′−h21xi+h22yi+h23h31xi+h32yi+h33)2\sum _i \left ( x'_i- \frac{h_{11} x_i + h_{12} y_i + h_{13}}{h_{31} x_i + h_{32} y_i + h_{33}} \right )^2+ \left ( y'_i- \frac{h_{21} x_i + h_{22} y_i + h_{23}}{h_{31} x_i + h_{32} y_i + h_{33}} \right )^2

如果参数 method 被设置为默认值 0 ，函数则使用所有的点对来计算一个简单的最小二乘法来计算初始 homography estimate

然而，如果不是所有的点组 (srcPoint_si, dstPoint_si) ，都符合 rigid perspective transformation(即存在一些异常值)，那么这个初始估计效果会很差。在这种情况下，可以使用三种健壮的方法其中一个。 RANSAC 、 LMeDS 和 RHO 方法尝试了许多不同的对应点组的随机子集(每四个一组，舍弃共线组)，利用这个子集和一个简单的最小二乘算法估计 homography matrix ，随后计算 computed homography 的质量/优劣(这是 RANSAC 的 inliers 数量或 LMeDS 的最小中位数的 re-projection error)。之后利用最佳子集产生的 homography matrix 的初始估计和 inliers/outliers 的掩码

无论该方法是否稳健，计算的 homography matrix 都会进一步使用 Levenberg-Marquardt 方法加以改进(仅在文件方法的情况下使用 inliers)，从而进一步减少 re-projection error

RANSAC 和 RHO 方法实际上可以处理任何比例的 outliers ，但是需要一个阈值来区分 inliers 和 outliers 。而 LMeDS 方法不需要任何阈值，但只有当超过 50 % 的 inlers 时才能正常工作。如果没有异常值且噪点相当小，则使用默认方法。 method=0

cv::perspectiveTransform

对向量进行透视矩阵变换

void cv::perspectiveTransform(src, det, m)

  • src：输入双通道或三通道浮点数组，每个元素都是一个需要转换的 2D/3D 向量
  • det：与 src 大小相同和类型相同的输出数组
  • m：3x3 或 4x4 浮点变换矩阵

该函数将 src 中的每个元素作为 2D 或 3D 向量进行变换，方法如下

(x,y,z)→(x′/w,y′/w,z′/w)(x, y, z) \rightarrow (x'/w, y'/w, z'/w)

其中

(x′,y′,z′,w′)=mat⋅[xyz1](x', y', z', w') = \texttt{mat} \cdot \begin{bmatrix} x & y & z & 1 \end{bmatrix}

同时

w={w′if w′≠0∞otherwisew = \fork{w'}{if \(w' \ne 0\)}{\infty}{otherwise}

这里显示的是一个三维向量变换。在二维矢量变换的情况下，省略 z 分量

注意：该函数对二维或三维向量的稀疏集合进行变化，如果你想用透视变换来变换图像，使用 warpPerspective 。如果你有一个 inverse problem ，即若想要从几组对应的点中计算最可能的透视转换，可以使用 getPerspectiveTransform 或 findHomography

cv::line

画一条连接两点的线段。函数绘制图像中 pt1 和 pt2 点之间的线段。这条线被图像边界所裁剪。对于整数坐标的 non-antialiased liens ，使用 8-connected 或者 4-connected Bresenham 算法。粗线以圆角结尾。Antialiased lines 是使用 Gaussian filtering 绘制的

void cv::line(img, pt1, pt2, color, thickness=1, lineType=LINE_8, shift=0)

  • img：在 img 图像上绘制
  • pt1：线段的第一个点
  • pt2：线段的第二个点
  • color：线的颜色
  • thickness；线的厚度
  • lineType；线的类型
  • shift；点坐标中的小数位数
