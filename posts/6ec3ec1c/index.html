Policy Gradient

Policy 就是给一个外界的输入，然后它会输出actor现在应该要执行的行为。一般写成 π ，如果用的是深度学习的技术来实现强化学习的话，policy 就是一个 network。在 network 里面就有一堆参数，用 θ 来表示 π 的参数

  • Input：现在机器所能看到的东西
  • Output：机器要采取什么样的行为

上图就是具体的例子：

  • policy 就是一个 network
  • input 就是游戏的画面，它通常是由 pixels 所组成的
  • output 就是能有几个动作可以选择去执行的， output layer 就有相应数量的 neurons

    假设现在可以做的行为有 3 个， output layer 就是有 3 个neurons。每个 neuron 对应到一个可以采取的行为。Input 一个东西后，network 就会给每一个可以采取的行为一个分数（概率）。actor 根据这个分数来决定要采取的行为。 比如 70% 执行 left，20% 执行 right，10% 开火等等。根据分布不同，actor 采取的行为就会不一样

Actor，Environment，Reward

一场游戏叫做一个 回合(Episode) 或者 试验(trial) ，所得到的 奖励(reward)总和就是回报(return)，用 R 表示。Actor就是要去最大化可以得到的奖励

Environment

Environment 可以看做是一个 function， 它一开始就会产生一个 state ，接下来 actor 会在这个 state 下产生一个 a1，然后 environment 把 a1 当做它的输入， 然后产生 s2，actor 会在 s2 下产生一个 a2 。这会一直持续下去，直到 environment 觉得要停止为止

在一场游戏里，我们把 environment 输出的 s 跟 actor 输出的行为 a 全部串起来，叫做一个 Trajectory：

每一个 trajectory，可以计算它发生的概率。假设现在 actor 的参数已经被给定，即 θ。根据 θ ，就可以计算某一个 trajectory 发生的概率，你可以计算某一个 episode :

  • 计算 environment 输出 s1 的概率

  • environment 根据 a1 和 s1 产生 s2，因为 s2 和 s1 是有关系的，即下一个游戏画面跟前一个游戏画面通常是有关系的，因为它们是连续的
  • 反复执行，直到结束，就可以计算一个 trajectory 出现的概率有多大。这个概率取决于两部分
    • environment 的行为
    • agent 的行为

Reward

Reward function 根据在某一个 state 采取的某一个 action 决定现在这个行为可以得到多少的分数，它是一个函数，输入 s 和 a 就可以得到对应的 r。将所有 r 加起来，记为 R(τ)，代表一个 trajectory τ 的 reward。我们要做的就是调整 actor 内部的参数 θ ，使得 R 的值越大越好。 R 是一个随机变量，所以应该在给定某一组参数 θ 的情况下，会得到的 R 的期望值是多少
