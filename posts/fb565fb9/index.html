NNI 使用和遇到的一些问题

介绍 NNI

NNI：NNI（Neural Network Intelligence）是微软的开源自动调参的工具。它是一个轻量但强大的工具包，帮助用户自动的进行特征工程、神经网络架构搜索、超参调优以及模型压缩。NNI管理自动机器学习（AutoML）的Experiment，调度运行由调优算法生成的Trial任务来找到最好的神经网络架构和/或超参，支持各种训练环境，如本机、远程服务器、OpenPAI、Kubeflow、基于K8S的FrameworkController（如，AKS等），DLWorkspace（又称DLTS），AML（Azure Machine Learning），AdaptDL（又称ADL），和其他的平台甚至混合模式。

NNI 操作

yaml 变量说明

  • authorName: 作者名字
  • experimentName: 实验名字
  • trialConcurrency: 是 trail 的并发数，这个需要根据自己的GPU数量设置，而不是下面的gpuNum！！！为什么呢，因为一个trail代表一次调参的过程，理解为用一种超参数在运行你的 train.py，并发数设为 x ，就有 x 个 trainer 在训练！
  • maxExecDuration: 实验的最长执行时间，超过该时间就停止
  • maxTrialNum: 实验次数
  • trainingServicePlatform: 训练平台，有三种选择：local, remote, pai ，默认 local 本地训练就可以
  • searchSpacePath: 搜索空间 json 文件的路径
  • useAnnotation: 是否使用注解
  • optimize_mode: 优化模型，有 maximize 和 minimize 两种方式，根据需要自行修改即可。例如，使用的标准是 acc ，则使用 maximize 方法，若使用的标准是 loss ，则使用 minimize
  • command: 是 NNI 的运行后将要执行的指令，mnist.py改为你的main.py或者train.py等等主程序
  • gpuNum: 是每个 trail 所需要的 GPU 个数，而不是整个 nni 调参所需要的 GPU 个数。对于大型任务，单独训练一次需要 N 个 GPU 的话，这个值就设置为N；如果单次训练，一个GPU 就足够，请把这个值设置为1

所以最后需要的 GPU 总数为 trialConcurrency gpuNum ，即 trail 的个数 每个 trail 需要的 GPU 个数

开启

切换到代码的目录下，直接运行。
-p代表使用的端口号。注意如果代码使用的是conda虚拟环境，需要激活conda虚拟环境。

停止 NNI

在你不想继续搜寻超参数的时候，可以使用 nnictl stop 停止自动调参。
但是这只是说不会开始下一个trail了，没有运行完的 trail 依然会继续执行。如果想终止 trail 可以用 nvidia-smi 中找到训练进程的 PID ，然后用 kill -9 PID 杀死进程

遇到的问题

ModuleNotFoundError: No module named ‘ruamel’

Win10 版本会，Ubuntu 版本并无这个问题

今天安装 NNI 遇到了： ModuleNotFoundError: No module named 'ruamel' 问题，该问题的解决方式如下：

  • 根据控制台输出：e:\anaconda\lib\site-packages\nni\tools\nnictl\common_utils.py ，修改 common_utils.py ，将 import ruamel.yaml as yaml 修改成 import ruamel_yaml as yaml
  • 退出保存
  • 继续执行 NNI git 上的验证安装的命令：nnictl create —config nni\examples\trials\mnist-pytorch\config_windows.yml
  • 发现又出现一样的错误，根据控制台的输出：e:\anaconda\lib\site-packages\nni\experiment\config\base.py ，修改 base.py ，import ruamel.yaml as yaml 修改成 import ruamel_yaml as yaml
  • 退出保存
  • 继续执行 NNI git 上的验证安装的命令：nnictl create —config nni\examples\trials\mnist-pytorch\config_windows.yml
  • 发现又出现一样的错误，根据控制台的输出：e:\anaconda\lib\site-packages\nni\tools\packageutils_init.py ，修改 init.py ，import ruamel.yaml as yaml 修改成 import ruamel_yaml as yaml
  • 退出保存
  • 继续执行 NNI git 上的验证安装的命令：nnictl create —config nni\examples\trials\mnist-pytorch\config_windows.yml
  • 成功执行

远程服务器启动 Web 页面

如果在服务器或远程计算机上启动 Experiment， 可将此 IP 替换为所连接的 IP 来查看 Web 界面，如 http://[远程连接的地址]:8080

【待验证】使用的是远程 GPU 服务器，网页打不开

解决方法：

  • 进行重定向：ssh -p remote_port -L 8888:127.0.0.1:8888 username@remote_ip
    • remote_port 是服务器端口号
    • 127.0.0.1 代表 localhost 可以相互替换，
    • 前面的 8888 是 -p 后面写的端口号，
    • 后面的 8888 是你要重定向到本机的端口号，可以随意填写
    • username是服务器的用户名，remote_ip是服务器的ip地址
    • 例如：ssh -p 8080 -L 8080:127.0.0.1:8888 ubuntu@100.20.65.33 或 ssh -L 8888:localhost:8888 ubuntu@100.20.65.33
  • 然后在自己电脑上打开浏览器，输入127.0.0.1:8888，就可以了
