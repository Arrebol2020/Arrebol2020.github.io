机器学习

  • Arthur Samurel: Field of study that gives computers the ability to learn without being explicitly programmed.

    在没有明确目标设置的情况下使计算机具有学习能力的研究领域

  • Tom Mitchell：A computer program is said to learn from experience E with repect to some task T and some performance measure P, if it’s performance on T as measured by P, improves with experience E.

    计算机程序从经验 E 中学习解决某一任务 T 进行某一性能度量 P ， 通过 P 测定在 T 上的表现因经验 E 而提高

监督学习算法

  • 监督学习(supervised learning)：教会计算机做某件事情
  • 无监督学习(unsupervised learning)：让计算机自己学习

监督学习

我们给定算法一个数据集，其中包括了正确答案。例如给定一个房价数据集，在这个数据集中的每个样本，我们都给出正确的价格，即这个房子实际卖价。算法的目的就是给出更多的正确答案，即给将要出售的房子进行估价。

回归问题(我们的目标时预测一个连续值输出)和分类问题(预测离散的输出)都属于监督学习

无监督学习

我们给定算法一个数据集，并没有将数据集的正确答案给算法，计算机不知道要那它来做什么，也不知道每个数据点究竟是什么，计算机只被告知这里有一个数据集，你能找到某种结构吗。

线性回归(linear regression)

监督学习的流程

假设函数(hypothesis function)

我们给定一个带有正确答案的数据集，学习算法从这个数据集中输出一个函数 h (hypothesis function, 假设函数)，h 函数是一个引导 x 到 y 的函数, h 函数最终会尽最大拟合数据集，然后在新数据上输出结果

代价函数(cost function)

平方差代价函数(square error cost function)， 是解决回归问题最常用的手段

代价函数记为：J(θ)，算法的目标就是为了最小化 J(θ)

梯度下降(gradient descent)

梯度下降可以将代价函数最小化，主要流程如下：

  • 初始化 θ (初始化为 0 / 采用随机值)
  • 不停地一点点改变 θ，来使得 J(θ) 减小，直到找到 J(θ) 的最小值或者局部最小值

  • 上图 θ 只有两个取值，是为了方便观看。通常情况下，需要同时更新 θ 的值。
  • α 的取值需要注意。当 α 值太小，会导致梯度下降非常慢；当 α 值过大，梯度下降可能会越过最低点，甚至可能无法收敛或者发散

即使学习率 α 固定，梯度下降也可以收敛至局部最小值。当接近局部最小值时，梯度下降速度会自动减缓，所以并不需要另外减小学习率 α (处理比较复杂的模型时，可以先将 α 设置大一点，当迭代到一定的次数后，再减小 α 的值可以加快训练速度) 。“Batch” Gradient Descent：每一步的梯度下降都是用了全部的训练集样本

多元线性回归(multiple features linear regression)

多元指有多个特征量或变量来预测y值

特征缩放(feature scaling)

特征缩放的目的：就是将输入特征的取值约束到相近的范围内(如 -1~1)如果能确保输入特征处在相近的范围，则梯度下降就可以更快地收敛。
方法：

    1. 将某一输入特征值除以该特征的最大值
    1. 均值归一化(mean normalization)：将某一输入特征值 x 减去均值 μ ，让 x 特征值具有 0 的平均值。
      • 即 (x - μ) / s，其中 s 可以为：最大值-最小值，或者 x 的标准差

学习率(learning rate)

可以通过绘制 J(θ) - 迭代的次数图来判断程序是否正常工作，学习率的设置是否合理。

判断是否收敛：

    1. 自动的收敛测试：如果 J(θ) 的变化量小于一个特定的值 ε ，则已经收敛(由于 ε 难以确定，并不推荐)
    1. 通过观察 J(θ) - 迭代次数图

只要学习率足够小，J(θ) 在每次迭代之后都会下降。但是如果太小的话，梯度下降算法的收敛过程就十分缓慢；如果太大，代价函数 J(θ) 可能不会再每次迭代都下降，甚至可能不收敛

正规方程与梯度下降对比

正规方程只适用于简单的类似于线性回归问题。

对于 m 个训练样本，n 个特征：

  • 梯度下降：
    • 需要选取学习速率 α
    • 需要迭代很多次
    • 在特征变量很多的情况下，运行的结果很好
  • 正规方程
    • 不需要选取学习速率 α
    • 不需要进行迭代
    • 为了求解参数 θ ，需要求解 X^TX 的逆矩阵，而实现逆矩阵计算代价为 O(N^3) ，所以当 n 非常大时，计算的速度会非常缓慢
  • X^TX 不可逆，即为奇异矩阵时，常见的原因：
    • 特征中存在多余的特征：如 x1 英尺为单位的房屋面积，x2 米为单位的房屋面积，因为这两个特征可以相互转换，导致 X^TX 不可逆
    • 特行的数量过多(m <= n)：解决方法删除一些特征，或者使用正则化(regularization)
